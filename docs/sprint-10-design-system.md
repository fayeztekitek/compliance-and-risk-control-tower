# Sprint 10 — Design System

**Goal:** Reusable enterprise design system

## Tasks

### Foundation
- [ ] Color palette CSS variables (primary, secondary, accent, semantic colors)
- [ ] Dark mode color tokens
- [ ] Typography scale (headings, body, small, code, font families)
- [ ] Spacing scale (4px grid)
- [ ] Border radius, shadow, opacity tokens
- [ ] Breakpoint definitions

### Component Library Expansion
- [ ] Cards: metric card, entity card, clickable card, card grid
- [ ] Tables: sortable headers, filterable columns, paginated footer, expandable rows, row actions
- [ ] Charts wrapper: consistent color mapping, legend, tooltip, responsive
- [ ] Forms: input types (text, number, select, multi-select, date), validation display, layouts (inline, stacked)
- [ ] Buttons: variants (primary, secondary, outline, ghost, danger), sizes (sm, md, lg), loading state, icon button, button group
- [ ] Notifications: toast (success, error, warning, info), banner, inline alert, notification center dropdown
- [ ] Modals: sizes (sm, md, lg, fullscreen), nested modal, confirm dialog, form modal
- [ ] Empty states: per entity, with illustration, with action CTA
- [ ] Loading states: skeleton per component type (card, table, chart, form)
- [ ] Error states: inline error, full-page error, retry button, error boundary fallback

### Accessibility
- [ ] Color contrast audit (WCAG AA minimum)
- [ ] Focus indicators
- [ ] ARIA labels on all interactive components
- [ ] Keyboard navigation (tab order, escape, enter, arrow keys)
- [ ] Screen reader announcements for dynamic content

### Tooling
- [ ] Storybook or Ladle component catalog
- [ ] Chromatic visual regression testing
- [ ] Figma design tokens sync (optional)

### Documentation
- [ ] Component usage guidelines
- [ ] Accessibility checklist
- [ ] Migration guide for existing pages
